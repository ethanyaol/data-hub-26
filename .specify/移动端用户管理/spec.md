# 功能需求文档：移动端用户与录音人管理服务

**功能分支**：`[mobile-users-001]`  
**创建时间**：2026-03-29  
**状态**：初稿  
**输入**：“移动端用户管理”功能

## 用户场景与测试 *(必填)*

### 用户故事 1 - 移动端全量用户安全检索与分页 (优先级: P1)

作为授权管理员（UAP管理员、运营管理员或运维管理员），我需要通过后台看到所有已注册访问大盘的“移动端用户”列表，并通过指定 ID、手机号、角色等复合条件快速过滤出对应用户。由于此列表存放了全员用户隐私及敏感信息，所以用户姓名、联系方式等字段必须有安全的保密性脱敏展现。

**优先级说明**：基于 C 端设备采集的人员规模庞大，必须拥有最基础的集中列表管理和复合条件过滤手段来支撑日常运营服务。

**独立测试**：可以不带有数据编辑的操作，只挂载显示用户列表、验证脱敏规则以及执行各类筛选器的组合验证。

**验收场景**：

1. **前提** 授权管理员进入移动端用户页，**当** 列表展现默认 10条/页分页的名单，**则** 可以清晰看到账号“手机号码”中间四位变为星号 `****` 脱敏，“姓名”进行对应的隐私规则脱敏打码。
2. **前提** 授权管理员展开搜索栏，**当** 输入多条件后点击查询，**则** 能够精准缩小分页范围。
3. **前提** 授权管理员调整分布器或跳转页面，**则** 页面动态刷新将无缝显示重排的对应区段真实数据量。

---

### 用户故事 2 - 用户账号的安全与生命周期干预 (优先级: P1)

作为授权管理员（UAP管理员、运营管理员或运维管理员），在接获移动端用户无法登录、密码丢失或账号涉及违规乱传数据的情况时，我可以在后台快速重置其密码、修改其身份信息或者强制停用该用户的使用权限。

**优先级说明**：核心风控和客诉排查依赖。失去生命周期管理将使问题账号不可控。

**独立测试**：可针对某一条用户记录进行的弹框修改资料、重置和封禁解封流程闭环测试。

**验收场景**：

1. **前提** 授权管理员点击编辑用户信息按钮，**当** 修改相关字段保存后，**则** 该用户资料在表中成功更新。
2. **前提** 授权管理员点击“重置密码”，**当** 二次输入新密码并确认同意后，**则** 用户的密码会在服务端自动更新为管理员设定的新口令。
3. **前提** 授权管理员判定账号违规或其他原因，**当** 点击“停用”并二次确认，**则** 对象账户将被标记为禁用，无法从客户端认证登入。

---

### 用户故事 3 - 指定用户的下辖采集团队与录音人维护 (优先级: P2)

作为授权管理员（UAP管理员、运营管理员或运维管理员），一个主移动端用户名下可能包含了多名真正干活的录音人。我需要深入到该指定用户的名下，查看并规范他们的录音人名册，并确保所有人员年龄必须符合要求规范才能参与数据提交。

**优先级说明**：此设计服务专门的以主从/多维度管理下挂人员层级的业务形态，在解决特定类型的合集工作时非常刚需。

**独立测试**： `userId` 相关的跳转、挂载组件渲染、录音人创建流程可单独脱离大盘测试。

**验收场景**：

1. **前提** 授权管理员尝试“新建录音人”，**当** 填写的“年龄”一栏小于 **14岁**，**则** 系统必须阻挡创建并给出警告。
2. **前提** 授权管理员修改录音人信息，**当** 保存后，**则** 其身份修改将完整挂靠在该原始移动端用户 ID 下。
3. **前提** 授权管理员点击删除最后一个或第一顺位录音人，**则** 平台会弹出强阻断警报提示不允许执行该动作，以实现防空库保护。

## 边缘情况

- **当用户输入无效的年龄筛选时会发生什么？** 系统将实时阻断失误键入，且由授权管理员操作时，一旦判定小于 14 会自动弹窗警告并清空无效值。
- **当管理员停用一个已经处于停用状态的账户时会发生什么？** 界面按钮的状态词应严格绑定当前账号状态（当前禁用状态下，只会提供“启用”按钮），不支持对相同状态进行二道覆写。

## 功能需求 *(必填)*

### 功能性需求

- **FR-001**: 必须在针对“录音人”新建、编辑和查询的所有入口，强制施加非特殊允许情况下的“年龄不低于14岁（`Age >= 14`）”合规性验证阻断。
- **FR-002**: 系统必须保证任意挂靠在移动端的用户下，永远至少持有一个录音人身份。新增移动端用户时，系统必须自动同步创建一个标记为「默认」且关联该用户的录音人，该录音人的昵称初始化为用户的姓名。
- **FR-003**: 移动端用户与其同步创建的录音人须保持姓名/昵称实时一致。管理员修改用户姓名时，系统应同步更新关联录音人的昵称位，并向管理员提供同步更新的反馈提示。
- **FR-004**: 同步创建的初始录音人实体不可被删除，仅支持信息编辑（除昵称外）。删除操作仅开放给后续手动增补的非核心录音人。

### 核心实体

- **移动端用户**: 主访问用户，包含核心信息（`登录账号`, `姓名`, `角色`, `端侧类型`, `端侧型号`, `创建时间`, `状态` [启用/停用]等）。
- **录音人**: 隶属于某 `移动端用户` 名下的生产人员子实体，拥有其独立的隐私采集属性（`nickname`, `gender`, `age` [>=14], `contact`, `growthLocation` 等）。

## 成功标准 *(必填)*

### 可衡量结果

- **SC-001**: 从总览列表进入某用户挂靠下的录音人管理的页面切换与加载呈现应满足耗时小于 1 秒（针对不劣于千兆外网的情况）的业务流转无缝体验。
- **SC-002**: 上述定义的所有信息查询过滤、人员数据改动能在一个列表面内不刷新无页面级重载（SPA 机制）内完成局部替换。
- **SC-003**: 安全测试用例必须覆盖 100% 姓名和手机号码在 HTTP HTML 响应中的脱敏或 DOM 层安全加密屏蔽。
- **SC-004**: 执行违规边界年龄录入测试（譬如13岁），所有端点反馈阻断率应当是 100%。

**Independent Test**: `userId` 相关的跳转、挂载组件渲染、录音人创建流程可单独脱离大盘测试。

**Acceptance Scenarios**:

1. **Given** 授权管理员尝试“新建录音人”，**When** 填写的“年龄”一栏小于 **14岁**，**Then** 系统必须阻挡创建并给出警告。
2. **Given** 授权管理员修改录音人信息，**When** 保存后，**Then** 其身份修改将完整挂靠在该原始移动端用户 ID 下。
3. **Given** 授权管理员点击删除最后一个或第一顺位录音人，**Then** 平台会弹出强阻断警报提示不允许执行该动作，以实现防空库保护。

## Edge Cases

- **What happens when a user types an invalid age filter?** 系统将实时阻断失误键入，且由授权管理员操作时，一旦判定小于 14 会自动弹窗警告并清空无效值。
- **What happens when an admin disables an already inactive account?** 界面按钮的状态词应严格绑定当前账号状态（当前禁用状态下，只会提供“启用”按钮），不支持对相同状态进行二道覆写。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 必须提供移动端用户的全局列表（包含ID、加密手机、脱敏姓名、设备信息等），且提供基于这些属性的双行展开合拢式字段组合搜索能力。
- **FR-002**: 必须为“登录账号（Phone 号码）”字段提供正则表达式格式的展示层遮蔽（`$1****$2`四位星号脱敏安全保护）。
- **FR-003**: 必须为操作面板提供重置当前账户密码的功能机制，要求管理员二次输入新密码并二次确认后，方可完成密码更新。
- **FR-004**: 必须提供用户账户状态禁用/启用的软删除流转机制，并在进行高危风险的“停用”操作时增加弹窗防误触。
- **FR-005**: 必须支持页面间的关联跳转，从移动端账户可以一对多链接进入该名下的 `录音人(Recorder)` 名单表盘。
- **FR-006**: 必须在针对“录音人”新建、编辑和查询的所有入口，强制施加非特殊允许情况下的“年龄不低于14岁（`Age >= 14`）”合规性验证阻断。
- **FR-007**: 系统必须保证任意挂靠在移动端的用户下，永远至少持有一个录音人身份。不可删除最后一个，也不可删去初识默认的第一序位录音人。新增移动端用户时，系统必须自动同步创建一个标记为「默认」且关联该用户的录音人，该录音人的昵称初始化为用户的姓名。
- **FR-008**: 移动端用户与其同步创建的录音人须保持姓名/昵称实时一致。管理员修改用户姓名时，系统应同步更新关联录音人的昵称位，并向管理员提供同步更新的反馈提示。
- **FR-009**: 同步创建的初始录音人实体不可被删除，仅支持信息编辑（除昵称外）。删除操作仅开放给后续手动增补的非核心录音人。

### 交互展示规范

- **下钻页面联动**: 当管理员进入某特定移动端用户的录音人详情列表时，必须能在顶端看到该管理对象的关键摘要。
- **筛选状态持久化**: 在移动端用户列表页执行任何搜索（手机号、姓名、状态）或翻页操作后，进入录音人详情并返回时，必须保留先前的筛选与搜索状态，不可重置为初始全量状态。

### Key Entities

- **Mobile User**: 主访问用户，包含核心信息（`登录账号`, `姓名`, `角色`,  `端侧类型`, `端侧型号`,  `创建时间`, `状态` [启用/停用]等）。
- **录音人 (Recorder)**: 隶属于某 `Mobile User` 名下的生产人员子实体，拥有其独立的隐私采集属性（`nickname`, `gender`, `age`[>=14], `contact`, `growthLocation` 等）。
